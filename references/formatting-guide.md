# 排版规范

> 来自 SOUL.md + 用户偏好。此文件由 content-匠人维护。

## 中文排版规范

### 字体体系

| 元素 | 字体 | 字号 | 加粗 |
|------|------|------|------|
| 一级标题 | 黑体 | 16pt | 是 |
| 二级标题 | 黑体 | 14pt | 是 |
| 正文 | 宋体 | 12pt | 否 |
| 注释/脚注 | 微软雅黑 | 9pt | 否 |
| 数据表格 | 宋体 | 10pt | 否 |
| 代码块 | Consolas/等宽 | 10pt | 否 |

### 段落格式

- 首行缩进：2字符（中文正文）
- 行间距：1.5倍
- 段前间距：0.5行
- 段后间距：0.5行

### 中文标点规范

- 使用全角中文标点：，。！？：；""''
- 数字与单位之间加空格：`32 GB`（非`32GB`）
- 中文与英文之间加空格：`这是 Apple 的产品`
- 英文专有名词保持原样：`iPhone`、`GitHub`
- 百分号紧贴数字：`82%`

### 文件命名

- 中文文件名：`报告-2026年Q2业绩分析.docx`
- 英文文件名：`report-q2-2026-analysis.docx`
- 禁止特殊字符：`/ \ : * ? " < > |`

### Markdown 输出规范

- 标题层级从 `##` 开始（`#` 留给文档总标题）
- 列表使用 `-` 而非 `*`
- 代码块标注语言：` ` `python` 而非 ` ` ` `
- 链接带标题：`[文本](URL "标题")`
